Modelo

¿Qué es?

El modelo de representación de los datos es una estructura que define cómo se organizan y almacenan los datos en la aplicación. En este caso, el modelo está centrado en una base de datos que almacena la información de los usuarios.

La estructura de la tabla "usuarios" es la siguiente:

id:

Clave primaria, autoincremental

Nombre:

varchar(50), nombre del usuario.

correo:

varchar(50), correo electrónico del usuario.

fecha_nacimiento:

date, fecha de nacimiento del usuario.

contrasena:

varchar(50), contraseña del usuario (almacenada de manera segura, idealmente encriptada)

genero:

varchar(100), género del usuario

imagen:

archivo de imagen, foto de perfil del usuario

Funcionalidades de la aplicación:

Edición de perfil:

Permite a los usuarios actualizar su información personal..

Eliminación de cuenta::

Permite a los usuarios eliminar su cuenta y toda la información asociada.

Visualización de perfil:

Muestra la información del usuario registrada en su perfil.

Registro de usuarios:

Permite a los usuarios crear una nueva cuenta proporcionando su nombre, correo electrónico, fecha de nacimiento, género, contraseña e imagen de perfil.

Inicio de sesión:

Autentica a los usuarios mediante su correo y contraseña.

Infraestructura para el almacenamiento y recuperación de datos:

Base de datos:

MySQL para el almacenamiento de los datos estructurados.

Servidor de aplicaciones:

PHP para manejar las solicitudes del usuario y la lógica del servidor.

Conexión a la base de datos:

Mediante PDO (PHP Data Objects) para una conexión segura y eficiente a la base de datos MySQL.

Gestión de imágenes:

Almacenamiento de imágenes en el sistema de archivos del servidor y referencias a las rutas de las imágenes en la base de datos.

Vista

¿Qué es?

El modelo de representación de los datos es una estructura que define cómo se organizan y almacenan los datos en la aplicación. En este caso, el modelo está centrado en una base de datos que almacena la información de los usuarios.

Representación de los datos:

Vista de registro:

Formulario HTML que permite a los usuarios registrarse proporcionando su información personal. Incluye campos para nombre, correo, fecha de nacimiento, género, contraseña e imagen de perfil.

Vista de inicio de sesión:

Formulario HTML que permite a los usuarios ingresar su correo y contraseña para acceder a su cuenta.

Vista de perfil:

Sistema de plantillas para la vista.

Vista de edición de perfil:

Formulario HTML que permite a los usuarios actualizar su información personal.

Vista de eliminación de cuenta:

Confirmación de eliminación de cuenta con un botón para confirmar la acción.

Controlador/span>

¿Qué es?

Componente que actúa como intermediario entre la Vista y el Modelo. Su función principal es recibir las entradas del usuario (como clics, entradas de teclado, etc.), procesarlas y determinar la respuesta adecuada, que generalmente implica interactuar con el Modelo y actualizar la Vista. El Controlador contiene la lógica de la aplicación y gestiona el flujo de datos entre el Modelo y la Vista, asegurando que la interfaz de usuario refleje el estado actual de los datos y que las acciones del usuario se traduzcan en actualizaciones del Modelo.

Identificación de eventos necesarios:

Registro de usuario:

Evento: Envío del formulario de registro.

Lógica: Validar los datos ingresados, encriptar la contraseña, almacenar los datos en la base de datos y guardar la imagen en el servidor.

Inicio de sesión:

Evento: Envío del formulario de inicio de sesión.

Lógica: Validar las credenciales del usuario, iniciar la sesión y redirigir al perfil del usuario.

NVisualización de perfil:

Evento: Solicitud de la página de perfil.

Lógica: Recuperar los datos del usuario desde la base de datos y mostrar la información en la vista de perfil.

Edición de perfil:

Evento: Envío del formulario de edición de perfil.

Lógica: Validar los datos actualizados, almacenar los cambios en la base de datos y actualizar la imagen de perfil si se proporciona una nueva.

Eliminación de cuenta:

Evento: Confirmación de eliminación de cuenta.

Lógica: Eliminar los datos del usuario de la base de datos y la imagen de perfil del servidor, y cerrar la sesión del usuario.
